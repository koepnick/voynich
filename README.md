# Voynich Manuscript
This is where my Voynich ramblings live

I'm also using this as a dumping ground for various Jupyter notebooks, scripts, and other goodies that the community at large might find useful. 

I'm always open to merge requests. Less so for the application logic itself, and more for the possibility of crowd sourcing some of the annotation work.

> Any bit of software that doesn't fit into the base project and is more complicated than a script should be containerized if possible. 

A longer term goal is to create yet another means of encoding VM data. While I believe that the standards put into place by the community are perfectly fine for transcoding and rendering, I believe that a more data-oriented foundation will be much more extendable and be able to store far more metadata. The idea is to have the annotators annotate without having to memorize multiple series of alpha-numeric characters for each glyph.
